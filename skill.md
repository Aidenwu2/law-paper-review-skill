# Law Paper Review - 法学论文多专家并行自查

## Description
法学论文投稿前多专家并行自查。模拟8位不同角色的专家同时审查论文，覆盖方法论、文献、写作、合规、期刊匹配、答辩模拟、实证复现和图表审美，最终汇总为一份综合审查报告。支持中文（CLSCI）和英文（SSCI）法学论文，尤其适合实证法学/法律经济学方向。

## Trigger
审查论文、论文自查、自查论文、帮我审稿、review paper、paper review、投稿前检查、论文检查、check my paper、审一下这篇论文

## Instructions

当用户要求审查法学论文时，按以下流程执行：

### 第一步：读取论文

用 Read 工具读取用户提供的论文文件（支持 PDF、DOCX、Markdown、纯文本）。如果用户没有提供文件路径，用 AskUserQuestion 询问。

### 第二步：判断论文类型

根据论文语言和内容，自动判断：
- **中文论文** → 按 CLSCI 标准审查
- **英文论文** → 按 SSCI 标准审查
- 如果不确定，询问用户

### 第三步：启动8位专家并行审查

用一条消息同时发出8个 Task 工具调用（`subagent_type: "general-purpose"`），每个 Task 的 prompt 必须包含完整的论文内容。

#### 专家1: 方法论审稿人 (Methodology Reviewer)
```
角色：资深实证法学方法论专家
审查维度：
- 研究设计是否合理（因果推断、内生性处理）
- 变量选取与操作化是否恰当
- 数据来源是否可靠，样本量是否充足
- 统计方法是否正确（回归模型选择、稳健性检验）
- 实证结果解读是否准确，是否过度推断
- 如为规范研究：论证逻辑链是否完整，是否有逻辑跳跃
输出格式：问题清单（严重/一般/建议），每条附具体位置和修改建议
```

#### 专家2: 文献审稿人 (Literature Reviewer)
```
角色：法学文献计量与综述专家
审查维度：
- 文献综述是否充分，是否遗漏该领域关键文献
- 中文文献：是否引用了法学研究、中国法学、中外法学等核心期刊的重要文章
- 英文文献：是否引用了该领域顶级期刊的经典和最新文献
- 文献引用是否准确（作者、年份、期刊是否对得上）
- 与已有研究的区分度是否清晰（创新点论证）
- 理论框架是否有坚实的文献支撑
输出格式：遗漏文献建议清单 + 引用问题清单 + 创新性评估
```

#### 专家3: 写作与结构审稿人 (Writing & Structure Reviewer)
```
角色：法学期刊资深编辑
审查维度：
- 标题是否准确反映论文核心贡献
- 摘要是否包含研究问题、方法、发现、结论
- 整体结构是否清晰合理
- 各部分篇幅比例是否恰当
- 论证逻辑是否连贯，段落之间是否有效衔接
- 学术语言是否规范，是否有口语化表达
- 脚注/尾注格式是否统一规范
- 中文论文：是否符合中文法学写作惯例
- 英文论文：语法、用词是否地道
输出格式：结构优化建议 + 具体语句修改建议（附原文和修改后对比）
```

#### 专家4: 学术合规审查员 (Academic Compliance Reviewer)
```
角色：学术诚信与出版伦理专家
审查维度：
- 是否存在论述过于接近某篇已有文献的风险
- 数据使用是否合规（伦理审查、知情同意等，如适用）
- 利益冲突披露是否完整
- 基金标注是否正确
- 是否存在自我抄袭风险（如与作者此前发表的文章高度重叠）
- 中文论文特别检查：政治敏感性、意识形态表述是否合规
- 参考文献列表是否完整、格式是否统一
输出格式：合规风险清单（高/中/低风险）+ 修改建议
```

#### 专家5: 期刊匹配顾问 (Journal Matching Advisor)
```
角色：法学出版与期刊策略专家
审查维度：
- 根据论文主题、方法、质量，推荐最匹配的期刊（3-5个）
- 中文论文：从 CLSCI 期刊中推荐，说明各期刊偏好
- 英文论文：从 SSCI 法学期刊中推荐，附影响因子和录用难度
- 分析论文与各推荐期刊的匹配度（高/中/低）
- 指出需要调整哪些内容以提高特定期刊的录用率
- 给出投稿顺序建议（先投哪个，被拒后转投哪个）
- 实证法学论文特别建议：是否考虑交叉学科期刊
输出格式：推荐期刊排名表（期刊名、匹配度、理由、需调整内容）
```

#### 专家6: 答辩模拟官 (Defense Simulator)
```
角色：资深法学教授，风格犀利
审查维度：
- 站在最挑剔的审稿人/答辩委员角度
- 提出 5-8 个最可能被质疑的问题
- 包括：核心概念界定、因果关系论证、替代解释、外部效度、政策建议可行性
- 对每个问题给出建议回应策略
- 指出论文最大的1-2个"硬伤"（如果有的话）
输出格式：模拟提问清单 + 建议回应 + 硬伤诊断
```

#### 专家7: 实证复现审查员 (Reproducibility Auditor)
```
角色：计量法学/法律实证研究复现专家
审查维度：
- 数据可获取性：数据来源是否公开可获取？是否提供了数据获取路径？
- 变量构造透明度：每个变量的操作化定义是否足够清晰，他人能否据此复现？
- 代码与流程：是否描述了完整的数据处理流程（清洗、编码、合并）？
- 模型可复现性：回归方程是否写清楚了？控制变量、固定效应、聚类标准误是否明确？
- 稳健性检验充分性：是否做了足够的稳健性检验（替换变量、子样本、替代模型）？
- 结果报告规范：表格是否报告了样本量N、R²、标准误类型、显著性水平？
- 如为规范/教义学研究：论证链条是否可追溯？法条、案例、学说引用是否可核实？
输出格式：
- 复现可行性评级：🟢 可复现 / 🟡 部分可复现 / 🔴 难以复现
- 具体问题清单（缺失信息逐条列出）
- 改进建议（补充哪些信息可提高复现性）
```

#### 专家8: 图表审美审查员 (Visual & Figure Reviewer)
```
角色：学术出版视觉设计专家，熟悉顶级法学/社科期刊的图表规范
审查维度：
- 图表必要性：每张图/表是否有存在的必要？是否有冗余？
- 表格规范：
  - 是否使用三线表（学术标准）？
  - 列对齐、小数位数是否统一？
  - 变量名是否清晰易懂（避免缩写不解释）？
  - 注释是否完整（数据来源、显著性标记说明、标准误类型）？
- 图片质量：
  - 分辨率是否达到期刊要求（通常 300dpi 以上）？
  - 字体大小在缩放后是否仍可读？
  - 配色是否专业（避免 Excel 默认配色、过于花哨的颜色）？
  - 黑白打印后是否仍能区分（部分期刊要求）？
- 图表标题与编号：
  - 标题是否自洽（不看正文也能理解图/表内容）？
  - 编号是否连续正确？
  - 正文中是否都有引用？
- 数据可视化选择：
  - 图表类型是否合适（该用柱状图的别用饼图，该用散点图的别用折线图）？
  - 坐标轴标签、单位是否标注？
  - 是否存在误导性的截断坐标轴或不等比例？
- 中英文差异：
  - 中文论文：图表中文字是否统一中文（或统一英文）？
  - 英文论文：图表中是否有残留中文？
输出格式：
- 图表总体评级：⭐⭐⭐⭐☆（满分5星）
- 逐张图/表审查意见（编号、问题、修改建议）
- 美观度改进建议（配色方案推荐、排版优化）
```

### 第四步：汇总综合报告

等待所有8位专家完成后，汇总为以下格式的综合报告：

```markdown
# 法学论文自查报告

## 论文信息
- 标题：xxx
- 类型：中文/英文
- 方向：xxx

## 总体评估
- 综合评分：⭐⭐⭐⭐☆（满分5星）
- 一句话评价：xxx
- 投稿就绪度：🟢 可以投稿 / 🟡 需要修改 / 🔴 需要大改

## 各专家审查摘要

### 1. 方法论审查
[核心发现摘要]
- 🔴 严重问题：x 个
- 🟡 一般问题：x 个
- 🟢 改进建议：x 个

### 2. 文献审查
[核心发现摘要]

### 3. 写作与结构
[核心发现摘要]

### 4. 学术合规
[核心发现摘要]

### 5. 期刊推荐
| 推荐排名 | 期刊名 | 匹配度 | 理由 |
|---------|-------|-------|------|
| 1 | xxx | 高 | xxx |
| 2 | xxx | 中 | xxx |

### 6. 答辩模拟
[最关键的3个问题]

### 7. 实证复现审查
- 复现可行性评级：🟢/🟡/🔴
[核心发现摘要]

### 8. 图表审美审查
- 图表总体评级：⭐⭐⭐⭐☆
[核心发现摘要]

## 优先修改清单（按紧急程度排序）
1. 🔴 [最紧急] xxx
2. 🔴 [紧急] xxx
3. 🟡 [建议] xxx
...

## 修改路线图
建议按以下顺序修改：
1. 先修复硬伤 → 2. 补充文献 → 3. 提高复现性 → 4. 优化结构 → 5. 美化图表 → 6. 润色语言 → 7. 格式检查
```

### 注意事项

1. **诚实为先**：如果论文有严重问题，必须直说，不要为了照顾情绪而淡化问题
2. **具体可操作**：每条建议必须具体到可以直接执行，不要给空泛的建议
3. **区分轻重**：用🔴🟡🟢三级标注问题严重程度
4. **实证特化**：因为用户偏实证法学方向，方法论审查要特别关注计量方法和因果推断
